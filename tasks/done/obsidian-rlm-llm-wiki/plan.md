## 📌 Core Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Plugin name | `obsidian-wiki-engine` | Shorter than `llm-rlm-obsidian-wiki-engine`, captures the LLM-native wiki concept |
| Migration strategy | Rename + retrofit `plugins/obsidian-integration` in place | Preserve all 6 existing skills, add 3 new ones |
| Raw content policy | **Multi-path manifest** (`raw-sources.json`) — no file moves required | User declares raw folders; tool indexes them without touching the originals |
| Output structure | **Rigid + opinionated** inside a user-nominated wiki root | `/wiki /rlm /meta` always in the same place under the wiki root |
| RLM integration | **Relative path calls into `rlm-factory/scripts/`** directly | Don't duplicate; leverage `distiller.py`, `inject_summary.py`, `query_cache.py` |
| Cheap work delegation | `gpt-5-mini` (or equivalent cheap model) | SKILL.md boilerplate, command YAMLs, template files, guided discovery, bulk classification, distillation |
| Architecture work | Orchestrating Claude session (Sonnet) | Script logic, plugin.json wiring, cross-skill integration |

## 🏗️ Vault Output Structure (Rigid)

User picks a `wiki-root` during onboarding. Everything below is rigid under it:

```
{wiki-root}/
  /wiki/              ← Karpathy-style wiki nodes (generated)
    _index.md         ← master concept index
    _toc.md           ← table of contents
    _{cluster}.md     ← per-topic cluster pages
    {concept}.md      ← individual wiki node
  /rlm/               ← RLM summary layers
    {concept}/
      summary.md      ← 1–5 sentence distilled summary
      bullets.md      ← key idea bullets
      deep.md         ← full multi-pass distillation
  /meta/              ← config, manifests, agent state
    raw-sources.json  ← list of raw content folders/files (user-declared via Guided Discovery Agent)
    config.yaml       ← wiki settings (depth, model, namespace)
    agent-memory.json ← processed file tracking (what's stale)
    embeddings.json   ← optional semantic search index
```

### Raw content stays where it is

`raw-sources.json` is generated by a **Guided Discovery Sub-Agent** during `wiki-init`. Example:
```json
{
  "namespace": "my-project",
  "wiki_root": "/Users/me/vaults/my-vault/wiki-root",
  "raw_sources": [
    { "path": "/Users/me/vaults/my-vault/notes", "label": "daily-notes" },
    { "path": "/Users/me/docs/architecture", "label": "arch-docs" },
    { "path": "/Users/me/research", "label": "research" }
  ],
  "excludes": ["_archive", "*.tmp"]
}
```

## ⚙️ Key Constraints & Logic Flow

### 1. The `dependencies.md` Requirement
At the root of the plugin, a `dependencies.md` file MUST be added.
**Content MUST include:**
Reminders that this plugin relies on `rlm-factory` being installed in the `.agents` folder, and the user must have Obsidian installed.
Must include exact installation command blocks:
```bash
uvx --from git+https://github.com/richfrem/agent-plugins-skills plugin-add richfrem/agent-plugins-skills
```

### 2. Guided Discovery Sub-Agent for Raw Content
During initialization (`/wiki-init`), a dedicated sub-agent guides the user to identify raw content directories rather than demanding they move files around. It asks questions to understand the layout and generates the `raw-sources.json` manifest transparently. 

### 3. Cost-Optimized Distillation Strategy (Strict Cheap-Model Fallback)
The `obsidian-rlm-distiller` skill (and `distill_wiki.py`) MUST operate entirely via sub-agents on **the cheapest possible model**, specifically avoiding slow models like local Ollama (which is being fully deprecated for distillation tasks in favor of `rlm-distill-agent`).

**Fallback Logic Prompt the user/agent must follow:**
1. Check if the user has **GitHub Copilot Pro**. If yes, automatically route distillation and bulk processing tasks to `gpt-5-mini` using `copilot-cli`.
2. If *not* Copilot Pro, but using **Claude Code** locally: fallback to `claude-3-5-haiku` (or `haiku-4.5`).
3. If using **Gemini CLI / Workspace**: fallback to `gemini-3.0-flash-preview` (or equivalent ultra-fast, cheap model).

**Never use `rlm-distill-ollama`. Only use `rlm-distill-agent` pointing to cheap models.**

## 🔄 Execution Order for Sonnet Session

1. **Heartbeat** — `copilot-cli-agent` heartbeat check (gpt-5-mini)
2. **Read context** — load `obsidian-plugin-bundle.md` + `rlm-factory-bundle.md`
3. **Rename** — `git mv plugins/obsidian-integration plugins/obsidian-wiki-engine`
4. **Copilot batch** — one `gpt-5-mini` call generates all SKILL.md, command YAMLs, templates, and `dependencies.md`
5. **Parse + write** — extract files from copilot output, write to disk
6. **Write core scripts** — including strict cheap-model fallback logic in `distill_wiki.py`
7. **Wire symlinks** — new skills point to shared scripts + rlm-factory
8. **Update plugin.json** — name, description, capabilities, full skills list
9. **Extend obsidian-init** — wire the Guided Discovery Sub-Agent for path identification
10. **Verify** — `find plugins/obsidian-wiki-engine -type l` to confirm all symlinks intact
