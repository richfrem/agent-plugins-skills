# Skill Ecosystem Mapping

The Exploration Cycle uses a hybrid capability execution model. We rely on a central set of **Native Skills** for Business Area Expert (BAE) exploration, and we rely on **Upstream `superpowers` Skills** for downstream technical execution.

## 1. Native to `exploration-cycle-plugin`
These skills are physically built, maintained, and deployed from our own `agent-plugins-skills/plugins/exploration-cycle-plugin/` repository. They orchestrate the Conversational Dashboard Pattern and support SME discovery.

| Skill / Node | Type | Purpose |
| :--- | :--- | :--- |
| `exploration-workflow` | Orchestrator | The primary SME entry point. Maintains the `exploration-dashboard.md` state machine and strictly routes the 4-phase loop. |
| `discovery-planning` | Phase 1 Agent | Interviews the SME to establish ground-truth requirements. Enforces the first `<HARD-GATE>`. |
| `visual-companion` | Phase 2 Agent | Validates layout and structural direction with the SME prior to technical build. |
| `subagent-driven-prototyping`| Phase 3 Agent | Orchestrates a multi-agent swarm to build functional, interactive UI prototypes for the SME to validate. |
| `exploration-handoff` | Phase 4 Agent | Consolidates all discovery, requirements, and process maps into a formal Spec package. Runs the "Scribe" activities. |
| `business-workflow-doc` | Scribe Node | Extracts future-state process maps interactively. (Invoked via Phase 4/Handoff) |
| `user-story-capture` | Scribe Node | Captures functional constraints and Agile stories. (Invoked via Phase 4/Handoff) |
| `exploration-optimizer` | Meta Node | Self-improvement loop to refine prompts and execution quality for the exploration cycle. |

*(Note: `exploration-orchestrator` and `prototype-builder` have been explicitly retired in the v2 Dashboard Pattern).*

## 2. Consumed from Upstream `superpowers`
These skills are natively provided by the `obra/superpowers` Github Copilot CLI extension. We **do not fork or modify** these skills. We simply consume them out-of-the-box for formal engineering execution (Opportunity 4) or when our Native Skills need to hand off work blocks.

| Skill / Node | Domain | Purpose |
| :--- | :--- | :--- |
| `brainstorming` | Strategy | Used by Copilot CLI to orchestrate design specs, checklists, and architecture plans. |
| `planning` | Engineering | Strict checklist generation for formal software engineering. |
| `implementing` | Engineering | TDD-based execution node that strictly adheres to the Spec package. |
| `debugging` | Engineering | Systematic root cause analysis of failures and broken tests. |
| `requesting-code-review` | SDLC | Final checks against the objective requirements before merging. |

## 3. The Boundary (Opportunity 3 vs Opportunity 4)
*   **Opportunity 3 (Exploration):** Exclusively uses the **Native Skills** (`exploration-workflow` + child skills). The SME never has to know that `superpowers` exists.
*   **Opportunity 4 (Engineering Execution):** Takes the formal output package generated by `exploration-handoff` (Phase 4) and feeds it directly into the **Upstream `superpowers` Skills** (`planning` -> `implementing` -> `requesting-code-review`) for deterministic SDLC execution.
