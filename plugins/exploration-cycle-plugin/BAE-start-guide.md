# BAE Quick Start — Guided Exploration Process

Purpose
-------
A concise how-to for Business Area Experts (BAEs) to start a guided Phase A exploration using the exploration-cycle-plugin. This guide keeps human-friendly, low-friction instructions and points to the interactive intake-agent as the recommended entry point.

When to use which entrypoint
----------------------------
- intake-agent (Recommended): high-touch interactive entry. Use for SME-led sessions where you want the assistant to ask follow-ups and pre-fill the session brief.
- exploration-workflow (Standard automation): use when you want the full Phase A loop run in a controlled sequence (intake → captures → prototype → handoff) and are comfortable with CLI-driven subagents.
- exploration-cycle-orchestrator-agent (Legacy/alternate): only use if automating at the orchestration-agent level or running pre-configured orchestrations. Prefer exploration-workflow for new sessions.

Quick start (interactive)
-------------------------
1. Open a terminal in your project workspace:
   cd /path/to/workspace # e.g. where exploration/ will live
2. Kick off the interactive intake (recommended):
   # This runs in your active Copilot session or primary agent context
   python3 .agents/skills/exploration-workflow/intake-agent.md

Quick start (CLI-driven automated Phase A)
-----------------------------------------
1. Run the exploration-workflow which orchestrates Phase 0–4:
   python3 .agents/skills/exploration-workflow/scripts/dispatch.py \
     --agent .agents/skills/exploration-workflow/intake-agent.md \
     --instruction "Run intake and start Phase A" \
     --output exploration/session-brief.md

2. Follow the prompts and run subsequent passes with dispatch.py using the examples in the exploration-workflow SKILL.md (problem-framing, brd-draft, user-stories).

Core artifacts and locations
----------------------------
- exploration/session-brief.md — session framing (created by intake-agent)
- exploration/captures/ — problem-framing.md, brd-draft.md, user-stories-draft.md, prototype-notes.md
- exploration/handoff/exploration-handoff.md — final handoff package
- assets/templates/exploration-dashboard.md — canonical dashboard template

Re-entry and scoped spikes
-------------------------
When engineering hits a blocker, create a small context file and re-run planning-doc-agent in re-entry mode:
  echo "Describe the blocker" > /tmp/reentry-ctx.md
  python3 .agents/skills/exploration-workflow/scripts/dispatch.py \
    --agent .agents/skills/exploration-workflow/planning-doc-agent.md \
    --context /tmp/reentry-ctx.md \
    --instruction "Mode: re-entry-scope" \
    --output exploration/session-brief-reentry.md

Best practices
--------------
- Use intake-agent for all BAE-led sessions — it maximizes funnel quality.
- Run gap checks after each capture pass (`check_gaps.py`).
- Keep captures iterative: baseline-first, change one variable at a time.
- All skills are active — there are no deferred/parked stubs.
