# /wiki-init

Initializes the Obsidian Wiki Engine for this project.

## What It Does

1. **Guided Discovery** — sub-agent interviews you to identify all raw content directories
2. **Generates `wiki_sources.json`** — the multi-source manifest (like `rlm_profiles.json` for RLM)
3. **Scaffolds wiki-root** — creates `/wiki`, `/rlm`, `/meta` directories
4. **Writes `config.yaml`** — default wiki settings
5. **Writes `agent-memory.json`** — empty stale-tracking ledger

## Usage

```bash
# Interactive (recommended first time)
/wiki-init

# With a wiki-root path
/wiki-init /path/to/my/vault/wiki-root
```

## Guided Discovery Sub-Agent

During init, a sub-agent will ask:

1. Where is your wiki root? (defaults to `{project}/.wiki`)
2. What raw content folders should be indexed? (repeat until done)
   - Path to folder?
   - Label for this source? (e.g. `daily-notes`, `arch-docs`)
   - File extensions to include? (default: `.md`)
   - Any subdirectories to exclude?
3. Confirm the generated `wiki_sources.json` before writing.

> **Raw files are never moved.** Only `wiki_sources.json` is created.
