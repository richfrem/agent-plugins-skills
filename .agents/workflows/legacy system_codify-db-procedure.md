# /codify-db-procedure

**Purpose:** Complete documentation workflow for Oracle Database Procedures. Delegates analysis to `/investigate-db-procedure`, then fills template and runs post-processing.


---

> **SYSTEM ENFORCEMENT PROTOCOL**
> 1. **MANDATORY EXECUTION**: You MUST invoke the `/legacy-system-database_investigate-db-procedure` skill via the Skill tool.
> 2. **PROHIBITED ACTION**: Manual XML reading, `grep`, or ad-hoc analysis is a VIOLATION.
> 3. **BLOCKING CONDITION**: If the investigation bundle is not generated by the skill, you MUST NOT proceed to documentation.
> 4. **FAILURE MODE**: If the tool fails, STOP and report the error. Do not fallback to manual work.

## Spec-Kitty Lifecycle Integration

This workflow follows the 6-step Spec-Kitty lifecycle:

| Step | Command | Description | This Workflow |
|------|---------|-------------|---------------|
| 1 | `/spec-kitty.specify` | WHAT to build | Phase 0 (auto) |
| 2 | `/spec-kitty.plan` | HOW to build | Phase 0 (auto) |
| 3 | `/spec-kitty.tasks` | Work packages | Phase 0 (single WP for codify) |
| 3.5 | `spec-kitty implement` | Create worktree | **Phase 0.5** |
| 4 | `/spec-kitty.implement` | Execute code | Phases 1-3 |
| 5 | `/spec-kitty.review` | Quality gates | Phase 3.5 |
| 6 | `/spec-kitty.merge` | Ship it | Phase 4 |

> **Note:** For simple documentation tasks, Phases 0-3 may be combined into a single `create-feature` call.

## Phase 1: Analysis (Call /investigate-db-procedure)

### Step 2: Run Investigation Module
```bash
/legacy-system-database_investigate-db-procedure [ProcedureName]
```

**What this does:**
1. Initializes context (`.agent/skills/context-bundler/scripts/manifest_manager.py init --target [ProcedureName] --type procedure`)
2. Runs the **Recursive Context Spiral** (`--direction downstream`, up to 3 levels)
3. Executes **Procedure Mining** (Signature, Tables, Validation Rules)
4. Performs **Dependency & Lineage Analysis** (`--direction both`: 1 level upstream, 3 levels downstream)

**Output:** Fully populated `temp/context-bundles/[ProcedureName]_context.md`

> **⚠️ DO NOT PROCEED until `/legacy-system-database_investigate-db-procedure` completes successfully.**

## Phase 1.5: Business Rule Reconciliation (Mandatory)
> **Objective:** Ensure we verify existing rules before creating duplicates.

1.  **Mined Logic Check**: Review `temp/context-bundles/[ProcedureName]_context.md` for mined logic (RAISE_APPLICATION_ERROR).
2.  **Reconciliation Loop**:
    - For each complex validation or logic block found:
    - `/legacy-system-business-rules_investigate-business-rule "[Keyword]"`
    - **IF FOUND**: Use the existing `BR-XXXX` ID.
    - **IF NEW**: Register it: `/legacy-system-business-rules_codify-business-rule --source [ProcedureName] --description "..."`

## Phase 4: Feature Completion — Deterministic Closure Protocol (MANDATORY)

> **Every step below is MANDATORY.** Skipping any step is a protocol violation.



Verify: in main repo root, clean status, no conflicts.

```bash

```

> **LOCATION RULE**: ALWAYS run merge from the **main repository root**.

```bash

```
Verify: merge visible, no orphaned worktrees, WP branches deleted, clean tree.
