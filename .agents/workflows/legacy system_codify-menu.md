# /codify-menu

**Purpose:** Generate comprehensive documentation for an Oracle Menu by calling the reusable `/investigate-menu` analysis module, then filling the Overview template.

> 
> **Architecture:** This workflow is an **OUTPUT WORKFLOW**. It calls the shared `/investigate-menu` module for analysis, then focuses on documentation output.

**Policy:** `plugins/legacy system/references/rules/menu_documentation_policy.md`
**Template:** `plugins/legacy system/assets/templates/forms-sops/sop-codify-menu-template.md`
**Diagram:** `../skills/legacy-system-oracle-forms/references/diagrams/workflows/menu-discovery.mmd`

---

> **SYSTEM ENFORCEMENT PROTOCOL**
> 1. **MANDATORY EXECUTION**: You MUST invoke the `/legacy-system-oracle-forms_investigate-menu` skill via the Skill tool.
> 2. **PROHIBITED ACTION**: Manual XML reading, `grep`, or ad-hoc analysis is a VIOLATION.
> 3. **BLOCKING CONDITION**: If `temp/context-bundles/[MenuID]_context.md` is not generated by the skill, you MUST NOT proceed to documentation.

## Phase 1: Analysis (Call /investigate-menu)

### Step 2: Run Investigation Module
```bash
/legacy-system-oracle-forms_investigate-menu [MenuID]
```

**What this does:**
1. Initializes context (`.agent/skills/context-bundler/scripts/manifest_manager.py init --target [MenuID] --type menu`)
2. Runs the **Recursive Context Spiral** (`--direction downstream`, up to 3 levels)
3. Executes **Multi-Pass Mining** (Hierarchy, Commands, Roles)
4. Performs **Security Verification** (`--direction both`: 1 level upstream, 3 levels downstream)

**Output:** Fully populated `temp/context-bundles/[MenuID]_context.md`

> **⚠️ DO NOT PROCEED until `/investigate-menu` completes successfully.**

### Step 4: Create or Update Overview
Create or Update `legacy-system/oracle-forms-overviews/menus/[MenuID]-Menu-Overview.md`.

**Rules:**
1. **Check for Collisions**: Does this ID exist as a PLL or Form? (e.g. ITSLIB). If so, ensure you clarify the Artifact Type (Menu) in the **Notes** section.
2. **Do Not Overwrite Blindly**: If the file exists, preserve unique manual insights (Business Rules, extra notes). Merge the new technical data (Hierarchy, Roles) into the existing file.

**Required Sections within the file:**
1. **Hierarchy**: Full menu tree.
2. **Role Mapping**: Which roles see which items.

**Output:** `legacy-system/oracle-forms-overviews/menus/[MenuID]-Menu-Overview.md`

```bash

spec-kitty merge --feature <SLUG> --dry-run
```
Verify: in main repo root, clean status, no conflicts.

```bash

```

> **LOCATION RULE**: ALWAYS run merge from the **main repository root**.
> NEVER `cd` into a worktree to merge.

Verify: merge visible, no orphaned worktrees, WP branches deleted, clean tree.
