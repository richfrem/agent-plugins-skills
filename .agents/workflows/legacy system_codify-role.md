# /codify-role

**Command:** `/codify-role [RoleName]`

**Purpose:** Create or Update the documentation for a specific User Role.
**Model:** Uses the same pattern as `/codify-business-rule` (Investigate -> Decide -> Act -> Sync).

**Template:** `plugins/legacy system/assets/templates/role-template.md`

**This is an ORCHESTRATING workflow.**

---

> **SYSTEM ENFORCEMENT PROTOCOL**
> 1. **MANDATORY EXECUTION**: You MUST invoke the `/legacy-system-roles_investigate-role` skill via the Skill tool.
> 2. **PROHIBITED ACTION**: Manual code reading, `grep`, or ad-hoc analysis is a VIOLATION.
> 3. **BLOCKING CONDITION**: If `temp/context-bundles/role_[RoleName]_context.md` is not generated by the skill, you MUST NOT proceed to documentation.
> 4. **FAILURE MODE**: If the tool fails, STOP and report the error. Do not fallback to manual work.

---

## Phase 0: Pre-Flight (The Gate)

```bash
spec-kitty agent feature create-feature "codify-role-[RoleName]" --json
```

**What this does:**
1.  **Context Check**: Verifies git branch and environment.
2.  **Spec Check**:
    - Checks for `kitty-specs/[ID]-[RoleName]/spec.md`.
    - **IF MISSING**: Orchestrator STOPS and prompts: *"No Spec found. Please run `/spec-kitty.specify` first."*
    - **IF PRESENT**: Validates `plan.md` and `tasks.md` exist.

> **⚠️ STOP:** Do not proceed to Phase 1 until `spec-kitty` returns SUCCESS.

## Phase 1: Investigation (Recursive)

### Step 1: Deep Dive
Execute the atomic, recursive investigation workflow.
```bash
/investigate-role [RoleName]
```
*   **Action:** Runs iterative mining loop.
*   **Output:** `temp/context-bundles/role_[RoleName]_context.md`

---
 
 ## Phase 1.5: Business Rule Reconciliation (Mandatory)
 > **Objective:** Ensure we verify existing rules before creating duplicates.
 
 1.  **Logic Check**: Review `temp/context-bundles/role_[RoleName]_context.md` for implied rules (e.g., "Access limited to...").
 2.  **Reconciliation Loop**:
     - `/legacy-system-business-rules_investigate-business-rule "[Keyword]"`
     - **IF FOUND**: Use the existing `BR-XXXX` ID.
     - **IF NEW**: You **MUST** register it using the creation workflow:
       ```bash
       /legacy-system-business-rules_codify-business-rule --title "[Role Rule]" --source "[RoleName]" --priority P2
       ```
       > **CRITICAL**: Do NOT manually create BR files. Use the workflow to ensure proper ID generation and inventory sync.
 
 ---

## Phase 2: Decision Matrix

Review the Output Bundle and determine the path:

**Outcome A: Role Exists (Active)**
*   **Condition:** Role found in `roles_inventory.json` and marked "Active".
*   **Action:** Proceed to **Phase 3: Update**. Use bundle to refresh capabilities.

**Outcome B: Role Missing (New/Unknown)**
*   **Condition:** Role NOT in `roles_inventory.json` but found in code.
*   **Action:** Proceed to **Phase 3: Create**. You are registering a previously undocumented role.
    *   *Note:* Add to `roles_inventory.json` via `../skills/legacy-system-roles/scripts/generate_role_inventory.py` (if tool exists) or manual edit.

**Outcome C: False Positive**
*   **Condition:** "Role" is actually a variable name or typo.
*   **Action:** Stop.

## Phase 3: Documentation (Scribe)

### Step 2: Create or Augment File (Resulting Artifact)
**Target:** `legacy-system/justin-roles/[RoleName].md`

> **CRITICAL INSTRUCTION:**
> **Do NOT overwrite** the entire file if it already exists.
> **Enrich** the existing content with new findings.
> **Preserve** manual edits or insights previously added.

1.  **Load Template:** `plugins/legacy system/assets/templates/role-template.md`
2.  **Fill Sections** (Using findings from `role_[RoleName]_context.md`):
    *   **Description:** Synthesize from code comments, inventory, and **Application Context** (e.g., "Used in [App List]").
    *   **Capabilities:** List privileges (`VISIBLE`, `ENABLED` checks).
    *   **Menu Access:** Tabulate findings Include **Parent Form** context.
    *   **Button/Report Access:** Tabulate findings. Include `Secure` and `Navigate` flags for Buttons.
    *   **Form Access:** List ALL Forms (explicit checks + Forms accessed via Menus/Buttons).
    *   **Missing Objects:** Check `legacy-system/reference-data/missing_objects.json` for any related artifacts marked as missing.

### Step 3: QA & Compliance
Confirm the file matches the standard template structure.
