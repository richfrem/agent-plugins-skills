# /codify-report

**Purpose:** Generate comprehensive documentation for an Oracle Report (RDF/XML) by calling the reusable `/investigate-report` analysis module, then filling the Overview template.

> 
> **Architecture:** This workflow is an **OUTPUT WORKFLOW**. It calls the shared `/investigate-report` module for analysis, then focuses on documentation output.

> **SYSTEM ENFORCEMENT PROTOCOL**
> 1. **MANDATORY EXECUTION**: You MUST invoke the `/legacy-system-oracle-forms_investigate-report` skill via the Skill tool.

## Phase 1: Analysis (Call /investigate-report)

### Step 2: Run Investigation Module
```bash
/legacy-system-oracle-forms_investigate-report [ReportID]
```

**What this does:**
1. Initializes context (`.agent/skills/context-bundler/scripts/manifest_manager.py init --target [ReportID] --type report`)
2. Runs the **Recursive Context Spiral** (`--direction downstream`, up to 3 levels)
3. Executes **Report Miner** (Structure, Queries, Triggers)
4. Performs **Dependency & Lineage Analysis** (`--direction both`: 1 level upstream, 3 levels downstream)

**Output:** Fully populated `temp/context-bundles/[ReportID]_context.md`

> **⚠️ DO NOT PROCEED until `/investigate-report` completes successfully.**

### Step 3: Verify Context Completeness

- [ ] Source XML


```bash

spec-kitty merge --feature <SLUG> --dry-run
```
Verify: in main repo root, clean status, no conflicts.

### Step 13: Merge from main repo
```bash

```

> **LOCATION RULE**: ALWAYS run merge from the **main repository root**.
> NEVER `cd` into a worktree to merge.

Verify: merge visible, no orphaned worktrees, WP branches deleted, clean tree.
